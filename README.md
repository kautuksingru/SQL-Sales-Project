# Retail Sales Data Analysis using SQL

## Project Overview

This project focuses on analyzing retail sales data using SQL Server to extract meaningful business insights and improve decision-making. The project covers the complete data analysis process, including data import, data cleaning, exploratory data analysis (EDA), and business problem-solving using SQL queries.

The main objective of this project is to understand customer purchasing behavior, product performance, sales trends, and business patterns through structured SQL analysis.

## Dataset Information

The dataset contains retail sales transaction records, including:

* Transaction Details
* Customer Information
* Product Details
* Quantity Purchased
* Payment Methods
* Sales Information

## Key SQL Concepts Used

* Database Creation & Table Management
* Data Import using BULK INSERT
* Data Cleaning & Handling Missing Values
* Duplicate Removal using CTE & ROW_NUMBER()
* Aggregate Functions (`SUM`, `AVG`, `COUNT`)
* `GROUP BY` & `HAVING`
* Window Functions (`ROW_NUMBER`, `RANK`, `LAG`)
* Dynamic SQL
* Common Table Expressions (CTE)
* Stored Procedures & Functions
* Business-Oriented SQL Analysis

## Business Problems Solved

* Identified top-selling products by category
* Analyzed customer purchasing patterns
* Detected repeat customers for retention analysis
* Compared monthly sales performance
* Identified slow-moving products
* Evaluated payment mode preferences
* Cleaned duplicate and missing data records

## Business Impact

The insights generated from this project can help businesses:

* Improve inventory management
* Identify high-demand products
* Increase customer retention
* Make data-driven marketing decisions
* Optimize sales strategies
